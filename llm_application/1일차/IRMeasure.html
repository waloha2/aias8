평가 지표
검색 시스템의 성능을 어떻게 측정할까?
Success / Reciprocal Rank
Precision / Recall
예시 설정
쿼리 q: "머신러닝 기초"
랭킹 결과 D = [doc₁, ..., doc₆]
K = 3 (상위 3개)
관련 문서 (Relevant)
비관련 문서
전체 관련 문서: 3개
(Doc B, Doc D, Doc E)
Rank(q, D) = 첫 번째 관련 문서의 위치
위에서부터 내려가며 처음 만나는 ● 의 순위
Doc A(1번) ✗ → Doc B(2번) ✓ 발견!
Rank(q, D) = 2
Success@K & Reciprocal Rank
"첫 번째 관련 문서를 얼마나 빨리 찾았는가?"
↑ 첫 관련 문서 = 위치 2
Success@3
Rank(q, D) = 2 ≤ 3 → 1
= 1 ✓
RR@3
1 / Rank(q, D) = 1 / 2
= 0.5
💡 MRR@K = 여러 쿼리에 대한 RR@K의 평균
→ 시스템 전체의 "첫 관련 문서 도달 속도" 측정
Precision@K & Recall@K
"상위 K개 중 관련 문서 비율" vs "전체 관련 문서 중 찾은 비율"
Ret(D, 3) = { Doc A, Doc B, Doc C }
Rel(D, q) = { Doc B, Doc D, Doc E }
Precision@3
|Ret ∩ Rel| / K = |{Doc B}| / 3 = 1/3
= 0.33
Recall@3
|Ret ∩ Rel| / |Rel| = 1 / 3
= 0.33
💡 Precision = 반환 결과의 정확도
💡 Recall = 관련 문서 발견 완성도

Success/RR = 첫 관련 문서의 위치 품질
Precision/Recall = 전체 결과의 양적 품질
→ 두 관점을 함께 봐야 검색 시스템을 정확히 평가할 수 있다!
요약
Success@K
관련 문서가 있긴 한가?
Rank(q,D) ≤ K → 1, else 0
RR@K / MRR@K
첫 관련 문서가 얼마나 높은가?
1 / Rank(q,D)
Precision@K
반환 결과가 정확한가?
|Ret∩Rel| / K
Recall@K
관련 문서를 다 찾았나?
|Ret∩Rel| / |Rel|
위치 품질 + 정확도 + 완성도
= 종합적인 검색 품질 평가
