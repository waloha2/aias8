BM25 이해하기

Best Match 25 — 검색 엔진의 핵심 랭킹 알고리즘

BM25는 검색어(query)에 대해 문서(document)의 관련성 점수를 계산하는 알고리즘입니다.
Google, Elasticsearch, Lucene 등 대부분의 검색 엔진에서 사용됩니다.

핵심 아이디어: 희귀한 단어가 문서에 많이 나타날수록 → 더 관련성 높은 문서!

예제 문서 컬렉션 (D)

위 5개 문서에서 "머신러닝"을 검색한다고 가정해봅시다.
BM25는 3가지 요소를 조합합니다:

1. IDF (역문서빈도)

단어가 얼마나 희귀한가?

흔한 단어 → 낮은 점수
희귀한 단어 → 높은 점수

2. TF Scoring (단어빈도)

문서에 단어가 몇 번 나오는가?

많이 등장 → 높은 점수
하지만 포화(saturation) 적용!

최종 BM25 점수

BM25 = IDF × TF Score

Step 1: IDF

Inverse Document Frequency — 단어의 희귀도

IDF(w) = loge(1 + (|D| − df + 0.5) / (df + 0.5))
|D| = 전체 문서 수  |  df = 해당 단어가 등장하는 문서 수
직관: "the", "is" 같은 흔한 단어는 거의 모든 문서에 있으므로 df ≈ |D| → IDF ≈ 0
반면 "양자컴퓨팅" 같은 희귀한 단어는 df가 작아 → IDF가 커집니다.

단어별 IDF 비교

|D| = 5 (전체 문서 5개)

Step 2: TF Scoring

Term Frequency with Saturation — 단어 빈도 점수

Score(w, doc) = TF · (k + 1) / (TF + k · (1 − b + b · |doc|/avgdl))

높을수록 TF의 영향이 천천히 포화

1이면 긴 문서에 강한 페널티, 0이면 길이 무시

TF Saturation 곡선

TF가 증가해도 점수는 무한히 올라가지 않고 포화됩니다.
이것이 순수 TF와 BM25의 핵심 차이입니다.

문서 길이 보정 (b 파라미터)

긴 문서는 단어가 자연스럽게 더 많이 등장하므로
b가 이를 보정합니다. b=0이면 보정 없음.

Final: BM25 Score

IDF × TF Score = 최종 관련성 점수

BM25(w, doc, D) = ScoreBM25(w, doc) × IDFBM25(w, D)
각 문서에 대해 IDF(전체 컬렉션 기준)와 TF Score(개별 문서 기준)를 곱해서 최종 점수를 구합니다.
검색어가 여러 단어인 경우, 각 단어의 BM25를 합산합니다.
