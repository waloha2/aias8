Core Difference

Cross-Encoder: Query와 Document를 함께 concatenate해서 하나의 BERT에 입력. 모든 layer에서 query-document 간 full attention이 발생하여 매우 풍부한 interaction을 포착.

DPR: Query와 Document를 별도의 encoder로 각각 독립적으로 encoding. 최종 representation의 dot product로만 유사도를 계산. Document embedding을 미리 계산(pre-compute)해 둘 수 있어 확장성이 뛰어남.

Why Cross-Encoder doesn't scale

Cross-Encoder는 각 (query, document) 쌍마다 BERT forward pass를 수행해야 합니다.

Q개의 query, D개의 document가 있으면 총 Q x D번의 inference가 필요합니다.

DPR은 document embedding을 한 번만 계산하면 됩니다. 이후 query가 들어올 때마다 query encoding (1회) + dot product (매우 빠름)만 필요합니다. 총 비용: D + Q번의 encoding + 빠른 벡터 검색.

Training Data Format

Both models use the same training data format:

⟨ qi, doci+, {doci,k-} ⟩
  • qi: i번째 query
  • doci+: query에 대한 positive (relevant) document
  • {doci,k-}: query에 대한 negative (irrelevant) documents 집합

Shared Loss Function (Negative Log-Likelihood)

L = - log   exp(score(qi, doci+)) ⁄ [ exp(score(qi, doci+)) + Σj=1..n exp(score(qi, doci,j-)) ]

이 loss는 본질적으로 softmax cross-entropy입니다:

  • Positive document의 score를 높이고
  • Negative document들의 score를 낮추도록 학습
  • Cross-Encoder에서 score = Rep(q, doc), DPR에서 score = Sim(q, doc)
